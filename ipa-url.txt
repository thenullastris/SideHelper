# SideInstaller — unsigned IPA source
#
# By default the "sign-sideinstaller" workflow signs the .ipa asset attached to
# this repo's LATEST RELEASE (latest prerelease when run on the beta channel),
# so cutting a new release is all that's needed to publish a new build.
#
# To pin a specific unsigned IPA instead, put its direct download URL on the
# first non-comment line below. Editing this line also re-triggers the workflow.
# Leave it blank to keep tracking the latest release. Override for a one-off run
# with the UNSIGNED_IPA_URL env var.
