# SideInstaller — certificate pool sources
#
# The "sign-sideinstaller" workflow signs the IPA once per certificate found
# across ALL of the sources listed below. List one source per line; blank lines
# and lines starting with '#' are ignored. Add or remove a line to add/drop a
# source. Override the whole list for a one-off run with the CERT_ZIP_URL env
# var (space/newline-separated).
#
# Two source kinds are supported (the two below have different layouts — the
# workflow handles both):
#
#   1) A direct .zip URL. Expected layout inside the zip (one folder per cert):
#        <Name>/<Name>.p12
#        <Name>/<Name>.mobileprovision
#        <Name>/password.txt        (optional — p12 password; falls back to P12_PASSWORD)
#      (may be nested under an extra top-level folder, e.g. certificates/<Name>/…)
#
#   2) A GitHub repository URL (https://github.com/<owner>/<repo>). Only the
#      certificate files are fetched (a sparse, blob-filtered clone), so large
#      unrelated assets in the repo are never downloaded. Same per-folder
#      layout as above; extra per-cert subfolders (e.g. NexStore/) are ignored.
#
# MERGE / DE-DUPLICATION: the same signing certificate often appears in more
# than one source (and, within a source, across several provisioning-profile
# variants). Each unique certificate is signed only ONCE — duplicates are
# detected by the leaf certificate's SHA-1 fingerprint, and the first source
# listed here wins. Order the lines to control which copy is kept.
